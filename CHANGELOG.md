# 1.5.1

### Добавлено

1.  Поддержка аудиоформата `.ec3` (E-AC3) в интерфейсе и при обработке потоков.
2.  Опция «Создавать отдельную подпапку для каждого файла» в модуле демуксинга для автоматической группировки извлеченных дорожек.

### Исправлено

1. Исправлена логика группировки файлов в модуле «Муксинг»: устранена ошибка, из-за которой аудио-контейнеры (например, `.mka`) ошибочно принимались за видео-файлы, что препятствовало корректной замене оригинальных дорожек.
2. Исправлен ложный отчёт об успехе при даунмиксе: теперь после завершения работы DEE проверяется фактическое создание выходного файла на диске.

# 1.5.0

### Добавлено

1. Поддержка аудио-контейнеров `.mka` в общем списке форматов и в модуле демуксинга.
2. Добавлена система управления временными файлами (`TempFileManager`): реализовано автоматическое отслеживание и очистка данных как при штатном выходе, так и при следующем запуске.
3. Добавлена функциональность прерывания работы скриптов: основная кнопка запуска теперь работает как переключатель (Выполнить/Остановить), позволяя безопасно останавливать активные дочерние процессы и автоматически удалять неполные выходные файлы.
4. Добавлены опции удаления исходного файла после обработки и прямой подмены оригинала (переименование результата в имя исходного файла) для модулей управления и замены потоков.
5. Добавлена поддержка навигации с помощью боковых кнопок мыши (кнопка «Назад») для возврата к предыдущим разделам интерфейса.
6. Автоматическая конвертация субтитров формата Timed Text и WebVTT в ASS при демуксинге.
7. Обогащение заголовков дорожек через pymediainfo при отсутствии данных от mkvmerge.
8. Различение дорожек с одинаковым языком: к субтитрам в дереве добавляется заголовок дорожки для уникальности.

### Исправлено

1. Устранено «состояние гонки» при массовой параллельной обработке: исправлены ошибки в менеджере настроек и подсистеме создания папок, из-за которых файлы могли сохраняться в некорректную директорию.
2. Исправлена логика исключения и замены дорожек: обеспечено корректное использование специфичных флагов `mkvmerge` для каждого типа потока и предотвращена «утечка» глав и тегов из исходных файлов.
3. Исправлены ошибки первого запуска `deew` и ошибки доступа при работе с путями, содержащими кириллицу.
4. Исправлен алгоритм замены потоков в контейнерах MKV: заменяемые дорожки сохраняют свою оригинальную позицию в файле и полностью наследуют все метаданные (язык, имя, флаги доступности и приоритета).
5. Исправлен выход элементов интерфейса за пределы окна при использовании очень длинных имен файлов и путей: реализовано их автоматическое сокращение в статус-баре и списках замен.

### Изменено

1. Оптимизировано отображение названия приложения в таскбаре Windows: удален информационный суффикс, теперь при наведении на иконку отображается только «K-Tools».
2. Оптимизирована архитектура приложения: внедрён паттерн проектирования Singleton для менеджера настроек и всех инфраструктурных раннеров (FFmpeg, FFProbe, mkvmerge и др.). Это устраняет их многократную инициализацию и решает проблему избыточных сообщений в логе.
3. Улучшена диагностика ошибок: теперь захватывается полный вывод `mkvmerge` (stdout и stderr) при сбоях.
4. Реализована автоматическая установка флагов «по умолчанию» и «принудительно» для всех дорожек, добавляемых через модуль муксинга из внешних файлов для их приоритетного выбора в плеерах.
5. Полностью переработан «Безопасный режим» для `deew`: теперь временные копии исходных файлов не создаются, что экономит место на диске и исправляет ошибки определения длительности потока.
6. Улучшена система защиты файлов: реализовано автоматическое разрешение конфликтов имен (добавление суффиксов `_1`, `_2`) и блокировка записи в файлы, занятые другими процессами.
7. Расширена поддержка кодека ALAC: реализовано сохранение в контейнер CAF (расширение `.alac`) для обеспечения совместимости с FFmpeg и внешними плеерами.
8. Интерфейс транскодирования теперь динамически скрывает несовместимые опции (например, упаковку в M4A для форматов, отличных от QAAC/AAC/ALAC).
9. Унифицирована обработка имен файлов (Path.stem) во всех модулях приложения для повышения стабильности сопоставления.
10. Улучшен интерфейс модуля демуксинга: реализована полноценная поддержка родительских чекбоксов для файлов, позволяющая массово выделять или снимать выбор со всех дорожек файла одним кликом.
11. Устранено замусоривание рабочих директорий лог-файлами eac3to: реализована автоматическая очистка файлов журнала (log*.txt) после завершения работы утилиты.
12. Категория «Муксинг» переименована в «Контейнеры» для упрощения навигации и соответствия общему стилю интерфейса.
13. Уменьшена длительность анимаций перехода между вкладками скриптов.
