# K-Tools

**K-Tools** — это программный комплекс для технической обработки медиафайлов. Приложение автоматизирует типовые задачи ремуксинга, транскодирования и демультиплексирования, используя консольные утилиты (FFmpeg, mkvmerge, eac3to) и предоставляя графический интерфейс на базе PyQt6.

---

## 🛠️ Глобальные настройки

Параметры, управляющие поведением приложения:

*   **Тема оформления**: Выбор между Темной, Светлой или Системной темами. Изменение вступает в силу после перезапуска.
*   **Максимум параллельных задач**: Количество файлов, обрабатываемых одновременно. Рекомендуется устанавливать значение, равное половине количества логических ядер процессора.
*   **Перезаписывать существующие файлы**:
    *   *Выключено (по умолчанию)*: Если файл с таким именем уже существует в папке назначения, к новому файлу будет добавлен суффикс `_processed`.
    *   *Включено*: Существующий файл будет перезаписан без подтверждения.
*   **Использовать автоматическую подпапку**: При активации результаты будут сохраняться в подпапку рядом с исходным файлом.
*   **Имя подпапки для результатов**: Название папки (по умолчанию `KTools_Result`), которая создается при включении автоматической подпапки.

## 📂 Описание инструментов

### 🎞️ Видео и Контейнеры

#### 1. Ремуксинг (Transmuxing)
Смена формата контейнера без перекодирования потоков (копирование через `-c copy`).
*   **Целевой формат**: Перепаковка в MKV или MP4.
*   **Удалить исходный файл**: Удаление оригинала после успешного завершения операции.
*   **Поведение**: Процесс происходит мгновенно, так как данные видео и аудио не изменяются, а только перемещаются в новую структуру заголовков.

#### 2. Очистка метаданных (Metadata Stripping)
Удаление глобальных тэгов, названий дорожек и служебных метаданных.
*   **Суффикс выходного файла**: Текст, добавляемый к имени (например, `_cl`).
*   **Удалить исходный файл**: Опция удаления оригинала.
*   **Поведение**: Использует параметр `-map_metadata -1`. Видео и аудио потоки переносятся в новый файл в исходном виде.

#### 3. Управление потоками (Stream Manager)
Редактирование состава дорожек внутри контейнера.
*   **Режим работы**:
    *   *Удалить выбранные*: Исключает отмеченные дорожки из финального файла.
    *   *Оставить только выбранные*: Экспортирует только те дорожки, которые отмечены в списке.
*   **Упаковать аудио в M4A**: Если в результате обработки остается только одна аудиодорожка, она записывается в контейнер .m4a (удобно для извлечения звука).
*   **Поведение**: Интерактивно анализирует файл через `ffprobe` и позволяет точно настроить структуру выходного файла без потери качества.

### 🎵 Аудио инструменты

#### 4. Транскодирование аудио (Audio Converter)
Преобразование аудиопотоков в различные форматы.
*   **Целевой формат**: Поддержка 15 форматов (QAAC, FLAC, WAV, AC3, EAC3, MP3, OPUS, DTS, WavPack и др.).
*   **Битрейт/Качество**: Установка качества для Lossy-форматов и степени сжатия для Lossless-файлов.
*   **Качество QAAC**: TVBR режим (0-127). Значение 127 обеспечивает максимально достижимое качество Apple AAC.
*   **Упаковать в контейнер (m4a)**: Упаковка потока в стандартный медиаконтейнер `.m4a` вместо «сырого» потока (ADTS).

#### 5. Даунмикс в Stereo (DEE)
Преобразование многоканального звука в Stereo 2.0.
*   **Формат вывода**: AC3 или Dolby Digital Plus (E-AC3).
*   **Битрейт**: Настройка от 128 до 640 kbps.
*   **Поведение**: Использует **Dolby Encoding Engine** для качественного понижения канальности с соблюдением стандартов Dolby.

#### 6. Декомпозиция каналов (Demuxing)
Демультиплексирование многоканального файла на отдельные моно-каналы.
*   **Поведение**: Работает через `eac3to`, на выходе создает набор моно WAV-файлов для каждого канала многоканального трека.

#### 7. Изменение скорости аудио (Time-Stretch)
Коррекция скорости аудиопотока для синхронизации с различной частотой кадров видео.
*   **Режим**: Пресеты для преобразования PAL (25.0 fps) → NTSC (23.976 fps) и обратно.
*   **Выходной формат**: Беспотерьные FLAC или WAV.
*   **Поведение**: Изменяет темп через `eac3to`, обеспечивая синхронность с видеорядом.

### 🏗️ Муксинг и Сборка

#### 8. Муксинг (Muxing)
Мультиплексирование видео, аудио и субтитров в единый мастер-файл MKV.
*   **Заголовок субтитров**: Имя дорожки, которое будет отображаться в плеере.
*   **Удалить лишние дорожки**: Если исходное видео содержит звуковые дорожки, они будут удалены, а их место займут внешние аудиофайлы из группы.
*   **Поведение**: Автоматически сопоставляет файлы в папке по их именам (без учета расширений) и собирает их в MKV через `mkvmerge`.

#### 9. Демуксинг (Track Extractor)
Массовое умное извлечение мультимедийных дорожек из контейнеров MKV/MP4.
*   **Фильтрация**: Встроенный интерактивный интерфейс с категоризацией дорожек (Видео, Аудио, Субтитры) и фильтрами по кодеку, языку и разрешению. С поддержкой глобального выделения потоков.
*   **Шаблонизатор**: Настройка имен выходных файлов на основе тегов для безопасной пакетной обработки множества файлов без конфликтов перезаписи.
*   **Поведение**: За один проход извлекает все выбранные потоки из контейнера с использованием высокоскоростного движка `mkvmerge`.

#### 10. Замена потоков (Stream Substitution)
Замена дорожек внутри существующего файла на внешние компоненты.
*   **Поведение**: Позволяет заменить дорожку (например, аудио или субтитры) на внешнюю по ее индексу, сохранив при этом все остальные элементы исходного контейнера.

## 🏗️ Техническая реализация
    
*   **Ядро**: Модульная архитектура на базе абстрактных классов (`AbstractScript`) с использованием паттерна **Template Method** для унификации процесса обработки файлов.
*   **Безопасность путей**: Глобальный переход на `pathlib` для кроссплатформенной и надежной работы с файловой системой. Реализована автоматическая защита исходных файлов от перезаписи.
*   **Производительность**: Централизованное управление расширениями файлов через иммутабельные коллекции (`frozenset`) для высокоскоростной фильтрации.
*   **Инверсия зависимостей**: Инфраструктурный слой (FFmpeg, mkvmerge runners) полностью отделен от бизнес-логики и настроек, что обеспечивает высокую тестируемость.
*   **Многопоточность**: Реализация через `QThreadPool` исключает блокировку интерфейса при выполнении тяжелых задач.
*   **Тестирование**: Проект охвачен обширной базой из **160+ unit и интеграционных тестов**, гарантирующих стабильность всех инструментов.
*   **Бинарники**: Программа ищет утилиты (`kt-ffmpeg`, `mkvmerge`, `eac3to`, `dee`, `qaac64`) в директории `bin/`.
